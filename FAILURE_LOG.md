# プロジェクト内での失敗の記録

この文書は、作業中に起きた失敗と再発防止策を残すためのログです。成功した変更の履歴は `spec.md`、進行中の課題は `NEXT_ACTIONS.md` に記録します。

## 記録ルール

- 小さなミスでも、再発する可能性があるものは記録する。
- 人ではなく、原因となった手順・確認不足・仕組みに焦点を当てる。
- 推測と確認済みの事実を分ける。
- 認証情報、個人情報、非公開URLなどの機密情報は書かない。
- 解決後も削除せず、状態と再発防止策を更新する。
- 同じ原因の失敗は新しい項目を乱立させず、既存項目へ追記する。

## 状態

- `未対応`: 原因または対策が未確定
- `対応中`: 修正・検証を進めている
- `解決済み`: 修正と確認が完了
- `保留`: 外部要因や判断待ち

## 失敗ログ

### F-001: 旧作業フォルダを完全に削除できなかった

- 発生日: 日付不明（2026年6月22日以前）
- 状態: 保留
- 事象: OneDrive内の旧 `trader` フォルダは中身を削除できたが、フォルダ本体を削除できなかった。
- 影響: 旧フォルダが残り、作業場所を取り違える可能性がある。
- 原因: CodexセッションまたはOneDriveがフォルダを使用中だった可能性がある。原因は未確定。
- 対応: 現在の正式な作業場所を `C:\trader` に統一した。
- 再発防止: 作業開始時にカレントディレクトリとGitリポジトリを確認し、OneDrive側の旧フォルダでは編集しない。
- 次の確認: 関連プロセスが終了した状態で旧フォルダを削除できるか確認する。

### F-002: Search Consoleを旧URLで先に登録した

- 発生日: 日付不明（2026年6月22日以前）
- 状態: 解決済み
- 事象: サイトURLの移行前に、旧URLをGoogle Search Consoleへ登録していた。
- 影響: 新URLで所有権確認とサイトマップ送信をやり直す必要が生じた。
- 原因: 公開URLの確定と検索サービス登録の順序が揃っていなかった。
- 対応: 新URL `https://webkabu-log.github.io/` を登録し、確認用HTML・metaタグを追加して `sitemap.xml` を送信した。
- 再発防止: ドメインまたは公開URLを確定し、canonical、OGP、Analytics、Search Console、サイトマップの対象URLを一覧で照合してから登録する。

### F-003: AI就活シリーズの前後リンクに不整合があった

- 発生日: 2026年6月17日以前
- 状態: 解決済み
- 事象: AI就活シリーズの前後リンクが、実際の公開順と一致していなかった。
- 影響: 読者がシリーズを順番に移動できない箇所があった。
- 原因: 記事追加時に、新規ページだけでなく前後両方のページをまとめて確認する手順が不足していた。
- 対応: `job-log-04.html` → `study-ai-jobhunting-01.html` → `job-log-05.html` の順に修正した。
- 再発防止: 記事追加時は「直前ページ・新規ページ・直後ページ」の3ページを一組で確認し、トップ、ロードマップ、サイトマップも同時に照合する。

### F-004: SEOメタ情報と公開状態に不整合があった

- 発生日: 2026年6月17日以前
- 状態: 解決済み
- 事象: 一部ページでOGP/Twitterメタタグが不足し、月次レポートでは `noindex` と canonical が併存していた。また、一部の `og:title` でサイト名が重複していた。
- 影響: 検索エンジンやSNSに、意図した公開状態・タイトルが伝わらない可能性があった。
- 原因: ページ単位でメタ情報を追加しており、全ページ横断の確認が不足していた。
- 対応: 不足タグ、重複タイトル、公開状態の矛盾を修正した。
- 再発防止: 公開前チェックに title、description、canonical、robots、OGP、Twitter Cardの横断確認を含める。

### F-005: 公開記事数の集計基準が曖昧だった

- 発生日: 2026年6月22日
- 状態: 解決済み
- 事象: トップページの公開記事数が38本と表示されていたが、番号付きの公開記事を機械集計すると、Day 1〜8、投資Study 01〜19、AI就活01〜10、AI活用メモ01〜06の合計は43本だった。
- 影響: サイト上の実績表示と実際の公開記事数が一致せず、更新のたびに別の数え方をする可能性があった。
- 原因: 公開記事へ含めるシリーズの定義が文書化されておらず、手動で加算していた。
- 対応: 公開記事数を番号付き4シリーズの合計と定義し、トップと関連文書を43本へ更新した。
- 再発防止: 新規記事の公開時にシリーズ別件数を集計し、トップ、仕様書、引き継ぎの数字を照合する。案内ページ、転送ページ、固定ページは公開記事数へ含めない。

## 新規記録テンプレート

以下をコピーして追記します。

```markdown
### F-XXX: 失敗を一文で表すタイトル

- 発生日: YYYY-MM-DD
- 状態: 未対応 / 対応中 / 解決済み / 保留
- 事象: 実際に起きたこと
- 影響: ユーザー、公開サイト、作業への影響
- 原因: 確認できた原因。推測の場合はその旨を明記
- 対応: 実施した修正や暫定対応
- 再発防止: 次回から行う具体的な確認や仕組み
- 次の確認: 未対応事項がある場合のみ記載
```
